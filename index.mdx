## What is Kairo?

Kairo is the intelligent core of the **Riwi Learning Platform** — an ecosystem built to ensure no student is left behind during Riwi's intensive 11-month coding bootcamp. By combining academic data from Moodle with soft skills assessments, Kairo generates fully personalized 4-week learning paths tailored to each coder's pace, learning style, and weak areas.

<Info>
  Kairo runs as a hybrid system: a **Node.js/Express** API gateway handles authentication, business logic, and orchestration, while a **Python/FastAPI** microservice handles all AI inference and data science workloads.
</Info>

## Key capabilities

<CardGroup cols={2}>
  <Card title="Adaptive Learning Plans" icon="brain">
    Generates custom 4-week study plans using Moodle performance data and soft skills profiles. Supports both interpretive (first-time) and analytical (weekly refresh) modes.
  </Card>
  <Card title="Focus Cards" icon="cards-blank">
    Six AI-generated study cards per coder, distributed as 2 High / 2 Medium / 2 Low priority, targeting specific struggling topics.
  </Card>
  <Card title="AI Trainer Chat" icon="message-bot">
    On-demand tutoring powered by Groq's LLM (llama-3.3-70b-versatile) with context-aware responses for bootcamp topics.
  </Card>
  <Card title="Risk Detection" icon="triangle-exclamation">
    Automated risk scoring flags coders with low autonomy and poor academic scores before failures occur.
  </Card>
  <Card title="TL Analytics Dashboard" icon="chart-line">
    Team Leaders see clan-level progress, individual risk flags, and AI-generated executive reports.
  </Card>
  <Card title="Evidence Submissions" icon="file-check">
    Coders submit files and links as proof of activity completion, tracked per learning plan.
  </Card>
</CardGroup>

## Tech stack

</CardGroup>

## User roles

Kairo serves two distinct user types:

| Role | Description |
|------|-------------|
| **Coder** | A student in the bootcamp. Completes the soft skills onboarding, receives AI-generated learning plans, interacts with the AI Trainer, and tracks activity progress. |
| **Team Leader (TL)** | A mentor overseeing one or more clans (Turing, Tesla, McCarthy). Monitors risk flags, reviews AI reports, sends feedback, and assigns exercises to groups. |

## Get started

<Steps>
  <Step title="Configure your environment">
    Set up `.env` files for both the Node.js gateway and Python AI service. See [Environment Setup](/setup/environment).
  </Step>
  <Step title="Initialize the database">
    Run the PostgreSQL schema and seed scripts against your Supabase instance. See [Database Setup](/setup/database).
  </Step>
  <Step title="Start both backends">
    Launch the Node.js orchestrator and Python AI brain simultaneously. See [Running the Platform](/setup/running).
  </Step>
  <Step title="Access the platform">
    Open the frontend at `http://127.0.0.1:5500/frontend` and log in or register as a Coder or Team Leader.
  </Step>
</Steps>
