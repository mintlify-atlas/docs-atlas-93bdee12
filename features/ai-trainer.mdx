# AI Trainer

The AI Trainer is Kairo's conversational interface for getting instant help with bootcamp topics. It is powered by Groq's inference API running the `llama-3.3-70b-versatile` model and is available from the **AI Trainer** page in the coder dashboard (`IAtrainer.html`).

In addition to the chat interface, the AI Trainer page is also where you view and navigate your active learning plan — completing days, accessing the daily exercise modal, and checking TL-provided resources for each topic.

## Asking a question

The chat interface sends your message to the Python AI service via `POST /chat/ask`.

### Request format

```json
{
  "message": "What is the difference between a LEFT JOIN and an INNER JOIN in SQL?",
  "coder_id": 42
}
```

| Field | Required | Description |
|-------|----------|-------------|
| `message` | Yes | Your question or prompt. Cannot be empty. |
| `coder_id` | No | Your user ID. Included for future context enrichment. |

### Response format

```json
{
  "reply": "An INNER JOIN returns only rows that have matching values in both tables. A LEFT JOIN returns all rows from the left table plus any matching rows from the right table — unmatched right-side columns come back as NULL. Use INNER JOIN when you only care about records that exist in both tables. Use LEFT JOIN when you need to keep all records from the primary table regardless of whether a match exists."
}
```

## Model configuration

The model used is configured via the `MODEL_NAME` environment variable on the Python service. If the variable is not set, it defaults to `llama-3.3-70b-versatile`.

```bash
MODEL_NAME=llama-3.3-70b-versatile
GROQ_API_KEY=your_groq_api_key
```

## System context

Every request to the AI Trainer includes the following system prompt:

> You are Kairo, an expert coding tutor at Riwi bootcamp. You help students with programming questions in a concise, motivating, and practical way. Always give a short answer first, then explain if needed. Use simple language. Max 3 paragraphs.

This means responses are:
- **Concise** — a direct answer comes first, with explanation only if needed
- **Practical** — focused on what you need to do, not academic theory
- **Bounded** — no more than 3 paragraphs per response

## Typical use cases

- Ask about a concept you encountered in your daily plan activity
- Request a plain-English explanation of a piece of code
- Get help debugging a query, function, or component
- Ask for an alternative approach to an exercise
- Clarify the difference between two related topics in the current module

<Tip>
  Specific questions get better answers. Instead of "explain SQL", ask "why does my GROUP BY query fail when I include a non-aggregated column that is not in the GROUP BY clause?" The more context you give Kairo, the more targeted the response.
</Tip>

## API endpoint

| Method | Path | Service |
|--------|------|----------|
| `POST` | `/chat/ask` | Python AI service (port 8000) |

The Node.js API does not proxy chat requests — the frontend connects to the Python service directly or through the configured `PYTHON_API_URL`.

<Note>
  The AI Trainer has a `max_tokens` limit of 600 per response and a temperature of 0.7. If the AI service is temporarily unavailable, the endpoint returns a `500` with the message `"AI tutor temporarily unavailable"` rather than an empty response.
</Note>
