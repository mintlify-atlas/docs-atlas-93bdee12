## Endpoint

```
POST /generate-focus-cards
```

**Tag:** Focus Cards

Generates exactly 6 smart study cards tailored to the coder's soft skill profile and their current struggling topics. Cards are distributed as 2 High / 2 Medium / 2 Low priority, matching the `priority_level_enum` values stored in `complementary_plans`.

This endpoint is called by the Node.js gateway — not directly by the frontend.

## Request Body

<ParamField body="coder_id" type="integer" required>
  The coder's user ID. Used to look up their `learning_style`, `problem_solving`, and `autonomy` scores from `soft_skills_assessment`.
</ParamField>

<ParamField body="module_id" type="integer" required>
  The module ID the coder is currently studying. Passed to the AI as context.
</ParamField>

<ParamField body="struggling_topics" type="string[]">
  List of topic names the coder is struggling with. If empty, defaults to `"general programming concepts"`. Defaults to `[]`.
</ParamField>

## Request Example

```bash
curl -X POST http://localhost:8000/generate-focus-cards \
  -H "Content-Type: application/json" \
  -d '{
    "coder_id": 3,
    "module_id": 4,
    "struggling_topics": ["JOINs", "índices", "transacciones"]
  }'
```

## AI Personalization

The AI receives the coder's soft skill profile from `soft_skills_assessment`:

- **`learning_style`** — shapes how the theory and practice sections are written
- **`problem_solving`** (1–5) — influences the depth of examples
- **`autonomy`** (1–5) — influences how guided the practice descriptions are

If no soft skills record exists for the coder, the AI falls back to defaults (`learning_style: "mixed"`, scores: `3/5`).

## Errors

| Status | Condition |
|--------|-----------|
| `500` | AI returned text that could not be parsed as valid JSON |
| `500` | Any other upstream error (Groq API timeout, Supabase connection failure, etc.) |

```json
{ "detail": "AI returned invalid JSON" }
```

<Warning>
  The service logs a warning if the AI returns fewer or more than 6 cards, but does not raise an error — it returns whatever the AI produced. Callers should validate `data.cards.length === 6` defensively.
</Warning>
