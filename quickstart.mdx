# Quick Start

<Warning>
  Both backend engines — Node.js and Python — must be running simultaneously for Kairo to function. Starting only one will result in failed AI requests or missing session data.
</Warning>

<Note>
  Kairo requires an active Supabase instance (PostgreSQL). You will need a Supabase project URL, anon key, and service key before completing setup. Create a free project at [supabase.com](https://supabase.com).
</Note>

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/cesarrios25/12345
    cd 12345
    ```
  </Step>

  <Step title="Configure the Node.js environment">
    Create a `.env` file in the `backend-node/` directory with the following variables:

    ```env
    # SERVER CONFIGURATION
    PORT=3000
    NODE_ENV=development

    # SECURITY & SESSIONS
    SESSION_SECRET=your_session_secret

    # DATABASE (POSTGRESQL - SUPABASE)
    DB_HOST=your_host
    DB_PORT=5432
    DB_NAME=your_name
    DB_USER=postgres.your_user
    DB_PASSWORD=your_password
    DATABASE_URL=your_database_url

    # SUPABASE CONFIG
    SUPABASE_URL=your_supabase_url
    SUPABASE_ANON_KEY=your_anon_key

    # FRONTEND CONNECTION
    FRONTEND_URL=http://127.0.0.1:5500/frontend

    # PYTHON CONNECTION
    PYTHON_API_URL=http://localhost:8000

    # SOCIAL AUTH - GITHUB
    GITHUB_CLIENT_ID=your_client
    GITHUB_CLIENT_SECRET=your_client_secret

    # SOCIAL AUTH - GOOGLE
    GOOGLE_CLIENT_ID=your_client
    GOOGLE_CLIENT_SECRET=your_client_secret

    # AUTHENTICATION SMTP
    SMTP_HOST=smtp.gmail.com
    SMTP_PORT=465
    SMTP_SECURE=true
    SMTP_USER=your_email
    SMTP_PASS=your_app_password
    SMTP_FROM="Kairo <your_email>"

    # API KEY AI
    RIWI_IA_KEY=your_ia_key
    ```
  </Step>

  <Step title="Configure the Python environment">
    Create a `.env` file in the `backend-python/` directory:

    ```env
    # PYTHON CONFIGURATION
    PORT=8000
    ENV=development

    # OPENAI-COMPATIBLE API (Groq)
    OPENAI_API_KEY=your_api_key

    # MODEL IA
    MODEL_NAME=your_model

    # SUPABASE CONFIG
    SUPABASE_URL=your_supabase_url
    SUPABASE_SERVICE_KEY=your_service_key

    # BACKEND NODE PORT
    NODE_URL=http://localhost:3000

    # FRONTEND PORT
    FRONTEND_URL=http://localhost:5500
    ```
  </Step>

  <Step title="Install Node.js dependencies">
    ```bash
    cd backend-node
    npm install
    ```
  </Step>

  <Step title="Set up the Python virtual environment and install dependencies">
    <Tabs>
      <Tab title="Linux / macOS">
        ```bash
        cd backend-python
        python3 -m venv env
        source env/bin/activate
        pip install -r requirements.txt
        ```
      </Tab>
      <Tab title="Windows">
        ```powershell
        cd backend-python
        python -m venv env
        env\Scripts\Activate.ps1
        pip install -r requirements.txt
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Start the Node.js orchestrator">
    Open a terminal in `backend-node/` and run:

    ```bash
    npm run dev
    ```

    Look for the following confirmation in the output before proceeding:

    ```
    🚀 KAIRO API GATEWAY STARTED SUCCESSFULLY
    ```
  </Step>

  <Step title="Start the Python AI service">
    Open a second terminal in `backend-python/` (with your virtual environment activated) and run:

    ```bash
    uvicorn main:app --host 0.0.0.0 --port 8000 --reload
    ```

    Look for the following confirmation before proceeding:

    ```
    INFO: Supabase client initialized
    ```

    Uvicorn will also print the local address (`http://0.0.0.0:8000`) once the service is ready.
  </Step>

  <Step title="Open the frontend">
    Open a third terminal in the repository root and serve the frontend on port 5500. The recommended approach is VS Code Live Server, or:

    ```bash
    npx serve -p 5500
    ```

    Then navigate to:

    ```
    http://127.0.0.1:5500/frontend
    ```
  </Step>
</Steps>
