# Architecture

Kairo is built on a hybrid microservice architecture. Three independent runtime layers communicate over HTTP: a Node.js API Gateway that orchestrates all business logic, a Python/FastAPI service that owns all AI computation, and a Vanilla JS frontend that serves both Coders and Team Leaders.

All persistent state lives in a PostgreSQL database hosted on Supabase with Row Level Security (RLS) enforced at the database layer.

## Service overview

| Layer | Technology | Port | Responsibility |
|---|---|---|---|
| API Gateway | Node.js / Express 5 | 3000 | Auth, sessions, business logic, user management, EJS rendering, proxy to Python |
| AI Engine | Python / FastAPI | 8000 | LLM inference (Groq), embeddings, plan generation, reports, AI chat |
| Frontend | Vanilla JS + Bootstrap 5 | 5500 | Multi-Page Application for Coders and Team Leaders |
| Database | PostgreSQL via Supabase | — | Persistent storage with Row Level Security |

## Backend services

<CardGroup cols={2}>
  <Card title="Node.js API Gateway" icon="server">
    **Port 3000 — System Orchestrator**

    Built on Express 5 with `express-session` and Passport.js. Handles all user-facing concerns: authentication (local, GitHub OAuth, Google OAuth), session management, SMTP email, file uploads via Multer, and EJS server-side rendering. Proxies AI requests to the Python service. Exposes a health check at `GET /api/health`.
  </Card>
  <Card title="Python FastAPI AI Engine" icon="cpu">
    **Port 8000 — AI Brain**

    Built on FastAPI 0.115 with Uvicorn. Runs LLM inference via the Groq API using the configured model (default: `llama-3.3-70b-versatile`). Generates sentence embeddings with `all-MiniLM-L6-v2` (384 dimensions) via `sentence-transformers`. Produces learning plans, focus cards, AI chat responses, exercises, and PDF reports. Exposes an interactive API spec at `/docs`.
  </Card>
</CardGroup>

## Node.js routes

The API Gateway registers the following route modules under `/api`:

| Mount path | Route file | Purpose |
|---|---|---|
| `/api/auth` | `authRoutes.js` | Login, registration, OAuth callbacks, password reset |
| `/api/coder` | `coderRoutes.js` | Coder dashboard data, learning plan retrieval |
| `/api/tl` | `tlRoutes.js` | Team Leader analytics, cohort management |
| `/api/ai` | `iaRoutes.js` | Proxy layer — forwards AI requests to the Python service |
| `/api/notifications` | `notificationRoutes.js` | In-app and email notification management |
| `/api` | `assignmentRoutes.js` | Assignment creation and submission |
| `/api/diagnostics` | `diagnosticRoutes.js` | Diagnostic assessments and results |
| `/api/exercises` | `exerciseRoutes.js` | Exercise delivery and submission |
| `/api/profile` | `profileRoutes.js` | User profile reads and updates |
| *(root)* | `basicRoutes.js` | Health check and base routing utilities |

## Python AI endpoints

The FastAPI service exposes the following endpoints:

| Method | Path | Description |
|---|---|---|
| `POST` | `/generate-plan` | Generates a personalized 4-week learning roadmap for a Coder |
| `POST` | `/generate-focus-cards` | Produces AI focus cards based on diagnostic and plan data |
| `POST` | `/chat/ask` | Handles a single-turn message to the AI trainer chat |
| `POST` | `/generate-report` | Generates a TL analytics report in PDF format |
| `POST` | `/generate-exercise` | Creates a custom coding exercise for a Coder |
| `GET` | `/health` | Returns service status, active model name, and embedding availability |

## Database

Kairo uses PostgreSQL provisioned through Supabase. The schema uses custom enumerated types — including `role_enum`, `risk_level_enum`, and `ai_agent_enum` — to enforce data integrity across user profiles, learning plans, and AI-generated artifacts. Row Level Security policies ensure users can only access their own data.

SQL schemas and DDL scripts are maintained in the `/database` directory of the repository.

## CORS configuration

<Info>
  The Python service builds its allowed origins list dynamically at startup. It always permits `http://localhost:3000` and `http://localhost:5500` / `http://127.0.0.1:5500`. If the `NODE_URL` or `FRONTEND_URL` environment variables are set to valid HTTP URLs, those origins are appended automatically. This means you can point the AI service at any Node.js or frontend deployment without hardcoding origins.
</Info>

The Node.js service applies a separate CORS policy: in development it allows `http://localhost:5500` and `http://localhost:5173`; in production it restricts origins to the value of `FRONTEND_URL`.
