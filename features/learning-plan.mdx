# Personalized learning plans

A learning plan is a 4-week adaptive curriculum generated by Kairo's Python AI service. Each plan is tailored specifically to you: it takes your soft skills profile, your current bootcamp module, and — after your first week — your actual performance data from Moodle.

Every plan spans **20 days** (5 days × 4 weeks) and contains two activities per day: one technical activity and one soft skill activity. Each day specifies an estimated time, an activity type, and a skill focus area.

<Note>
  Only one plan is active at any given time. When a new plan is generated, Kairo deactivates all previous plans for your account before saving the new one (`deactivate_plans` is called on the Python service before persisting).
</Note>

## Plan types

<CardGroup cols={2}>
  <Card title="Interpretive plan" icon="seedling">
    Generated on your **first login**, immediately after you complete the onboarding diagnostic. Uses only your soft skills profile — no Moodle data is available yet.

    `plan_type: "interpretive"`

    Best for: getting started in week 1 of a new module with no prior performance history.
  </Card>
  <Card title="Analytical plan" icon="chart-line">
    Generated each week by the Monday scheduled job. Uses your Moodle progress data — your average score, the topics you are struggling with, and the weeks you have already completed.

    `plan_type: "analytical"`

    Best for: ongoing adaptation once you have performance history in the current module.
  </Card>
</CardGroup>

## What a plan contains

Each plan is stored as JSONB in the `plan_content` column of the `complementary_plans` table. The generated content includes:

- **Activities per day** — a technical activity and a soft skill activity for each of the 20 days
- **Estimated time** — the expected duration in minutes for each activity
- **Activity type** — one of `guided`, `semi_guided`, or `autonomous`, matched to your `autonomy` score
- **Skill focus** — the specific topic or soft skill dimension targeted by the activity
- **Targeted soft skill** — the single soft skill dimension the overall plan is designed to strengthen

## Priority system

Plans are assigned a priority level using the `priority_level_enum` type (`low` / `medium` / `high`). This value is stored in the `priority_level` column of the `complementary_plans` table and controls how urgently the plan is surfaced in the AI Trainer view.

```sql
CREATE TYPE priority_level_enum AS ENUM ('low', 'medium', 'high');
```

## Requesting a plan

Plans are generated by sending a `POST` request to the Python AI service at `/generate-plan`. The Node.js backend assembles the payload from your session and Moodle progress data before forwarding it.

### Interpretive plan request

```python
{
  "coder_id": 42,
  "module_id": 3,
  "plan_type": "interpretive",
  "current_week": 1,
  "average_score": 0.0,
  "struggling_topics": [],
  "weeks_completed": []
}
```

### Analytical plan request

Analytical mode adds three extra fields drawn from your `moodle_progress` row:

```python
{
  "coder_id": 42,
  "module_id": 3,
  "plan_type": "analytical",
  "current_week": 3,
  "average_score": 72.5,
  "struggling_topics": ["async/await", "closures"],
  "weeks_completed": [
    {"week": 1, "average_score": 68.0, "struggling_topics": ["promises"]},
    {"week": 2, "average_score": 75.0, "struggling_topics": ["closures"]}
  ]
}
```

| Field | Interpretive | Analytical | Source |
|-------|:---:|:---:|--------|
| `coder_id` | Yes | Yes | Session |
| `module_id` | Yes | Yes | `users.current_module_id` |
| `plan_type` | `"interpretive"` | `"analytical"` | Determined by cron or first login |
| `current_week` | `1` | Current week | `moodle_progress.current_week` |
| `average_score` | `0.0` | Actual score | `moodle_progress.average_score` |
| `struggling_topics` | `[]` | Topic names | `moodle_progress.struggling_topics` |
| `weeks_completed` | `[]` | History array | `moodle_progress.weeks_completed` |

## Generation flow

When the Python service receives a plan request it:

1. Validates the `plan_type` value
2. Fetches your soft skills profile from the `soft_skills_assessment` table
3. Fetches the module and its weekly structure
4. Deactivates any currently active plan for your account
5. Builds the AI context (adds weekly history for analytical plans)
6. Calls the LLM to generate the plan content
7. Saves the plan to `complementary_plans` and logs the generation to `ai_generation_log`

<Note>
  Plan generation is fire-and-forget from the Node.js side. When you complete the onboarding diagnostic, the API returns a `201` response immediately with `planRequested: true`. The plan is saved in the background — your dashboard polls for it automatically.
</Note>
