## Overview

All coder endpoints require:

1. **An active session** — the `riwi.sid` cookie set by a successful login or OTP verification.
2. **Role `coder`** — requests from `tl` accounts receive `403 Forbidden`.
3. **Completed onboarding** — if `firstLogin` is still `true`, the gateway returns `403` with `redirect: '/onboarding'` until the diagnostic assessment is finished.

**Base URL:** `http://localhost:3000`

All endpoints are mounted under `/api/coder`.

## Dashboard

### GET /api/coder/dashboard

Returns everything needed to render the coder's dashboard in a single request: profile, soft skills assessment, Moodle progress, active plan summary, TL feedback (last 5), active risk flags, and recent performance tests.

```bash
curl http://localhost:3000/api/coder/dashboard \
  -b cookies.txt
```

## Learning Plans

### GET /api/coder/plan

Returns the full active AI-generated complementary plan including all 20 days and the coder's completion state.

```bash
curl http://localhost:3000/api/coder/plan \
  -b cookies.txt
```

<ResponseField name="hasPlan" type="boolean">
  `false` when no active plan exists.
</ResponseField>

<ResponseField name="plan" type="object | null">
  Full plan detail when `hasPlan` is `true`.
  <Expandable title="plan fields">
    <ResponseField name="plan.id" type="number">Plan ID.</ResponseField>
    <ResponseField name="plan.moduleId" type="number">Module this plan targets.</ResponseField>
    <ResponseField name="plan.planContent" type="object">AI-generated daily content (20 days).</ResponseField>
    <ResponseField name="plan.targetedSoftSkill" type="string">Soft skill being developed.</ResponseField>
    <ResponseField name="plan.generatedAt" type="string">ISO generation timestamp.</ResponseField>
    <ResponseField name="plan.completedDays" type="object">Map of `{"1": {completedAt: ...}, ...}` for completed days.</ResponseField>
    <ResponseField name="plan.currentDay" type="number">First incomplete day number (1–20).</ResponseField>
    <ResponseField name="plan.completedCount" type="number">Total completed days.</ResponseField>
    <ResponseField name="plan.isComplete" type="boolean">`true` when all 20 days are done.</ResponseField>
  </Expandable>
</ResponseField>

```json 200 — active plan
{
  "hasPlan": true,
  "plan": {
    "id": 17,
    "moduleId": 4,
    "targetedSoftSkill": "time_management",
    "generatedAt": "2025-11-05T12:00:00Z",
    "completedDays": {
      "1": { "completedAt": "2025-11-06T09:00:00Z" }
    },
    "currentDay": 2,
    "completedCount": 1,
    "isComplete": false
  }
}
```

```json 200 — no plan
{ "hasPlan": false, "plan": null }
```

### POST /api/coder/plan/request

Triggers AI plan generation for the coder. The request returns immediately; plan generation happens asynchronously via the Python AI service.

**Request body**

<ParamField body="plan_type" type="string">
  Plan type to generate. Defaults to `interpretive`.
</ParamField>

<ParamField body="current_week" type="number">
  Current week number within the module. Defaults to `1`.
</ParamField>

```json 200
{
  "requested": true,
  "message": "Plan en generación. Reintenta en unos segundos."
}
```

### POST /api/coder/plan/:planId/day/:day/complete

Marks a specific day in the active plan as completed.

**Path parameters**

<ParamField path="planId" type="number" required>
  ID of the active plan.
</ParamField>

<ParamField path="day" type="number" required>
  Day number to mark complete. Must be between `1` and `20`.
</ParamField>

<ResponseField name="success" type="boolean">`true` on success.</ResponseField>
<ResponseField name="completedDays" type="object">Updated map of all completed days.</ResponseField>
<ResponseField name="completedCount" type="number">Total days completed so far.</ResponseField>
<ResponseField name="nextDay" type="number | null">Next incomplete day, or `null` if the plan is finished.</ResponseField>
<ResponseField name="isComplete" type="boolean">`true` when all 20 days are done.</ResponseField>

```json 200
{
  "success": true,
  "completedDays": {
    "1": { "completedAt": "2025-11-06T09:00:00Z" },
    "2": { "completedAt": "2025-11-07T10:30:00Z" }
  },
  "completedCount": 2,
  "nextDay": 3,
  "isComplete": false
}
```

```json 400
{ "error": "Día inválido. Debe ser entre 1 y 20." }
```

```json 404
{ "error": "Plan no encontrado o inactivo." }
```

## Exercises

### POST /api/coder/exercise/generate

Generates a coding exercise for a specific day in the coder's plan. Returns a cached exercise if one already exists for that plan/day combination; otherwise calls the Python AI service.

**Request body**

<ParamField body="planId" type="number" required>
  ID of the active complementary plan.
</ParamField>

<ParamField body="dayNumber" type="number" required>
  Day number within the plan (1–20).
</ParamField>

<ParamField body="topic" type="string" required>
  Topic for the exercise (e.g. `"SQL joins"`).
</ParamField>

<ParamField body="weekNumber" type="number">
  Week number within the module. Defaults to `1`.
</ParamField>

<ParamField body="description" type="string">
  Additional context or instructions for the AI.
</ParamField>

<ParamField body="difficulty" type="string">
  `beginner`, `intermediate`, or `advanced`. Defaults to `intermediate`.
</ParamField>

<ParamField body="moduleName" type="string">
  Module name passed to the AI for context. Defaults to `"Bases de Datos"`.
</ParamField>

<ParamField body="language" type="string">
  Programming language (e.g. `"python"`, `"sql"`). `null` lets the AI decide.
</ParamField>

<ResponseField name="success" type="boolean">`true` on success.</ResponseField>
<ResponseField name="exercise" type="object">Generated exercise object with `title`, `description`, `language`, `starter_code`, `hints`, `topic`, `difficulty`, `expected_output`.</ResponseField>
<ResponseField name="cached" type="boolean">Present and `true` when a cached exercise was returned.</ResponseField>

```json 200
{
  "success": true,
  "exercise": {
    "id": 88,
    "title": "SQL INNER JOIN Challenge",
    "description": "Write a query that joins the orders and customers tables.",
    "language": "sql",
    "starter_code": "SELECT ... FROM orders o ...",
    "hints": ["Use INNER JOIN", "Match on customer_id"],
    "topic": "SQL joins",
    "difficulty": "intermediate",
    "expected_output": "List of orders with customer names"
  },
  "cached": true
}
```

### GET /api/coder/resources

Lists PDF resources uploaded by the clan's TL. Optionally filtered by module.

**Query parameters**

<ParamField query="moduleId" type="number">
  Filter resources by module ID.
</ParamField>

<ResponseField name="resources" type="array">

    <ResponseField name="title" type="string">Resource title.</ResponseField>
    <ResponseField name="file_name" type="string">Original file name.</ResponseField>
    <ResponseField name="preview_text" type="string">Extracted text preview.</ResponseField>
    <ResponseField name="uploaded_at" type="string">Upload timestamp.</ResponseField>
    <ResponseField name="module_id" type="number">Associated module.</ResponseField>

### GET /api/coder/resource/:id/download

Generates a temporary signed URL (valid for 1 hour) for downloading a resource PDF.

**Path parameter**

<ParamField path="id" type="number" required>
  Resource ID. Must belong to the coder's clan.
</ParamField>

```json 200
{
  "url": "https://storage.example.com/signed?token=...",
  "fileName": "module4-week3-guide.pdf"
}
```

```json 404
{ "error": "Recurso no encontrado o no autorizado." }
```
