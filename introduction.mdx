# Introduction

Kairo is the intelligent backbone of Riwi's intensive 11-month coding bootcamp. It bridges the gap between a fast-paced curriculum and each learner's individual pace by transforming academic performance data and soft skills assessments into dynamic, personalized 4-week growth paths.

At its core, Kairo is a distributed system: a Node.js orchestrator handles all business logic, authentication, and user management, while a Python/FastAPI microservice runs local LLM inference and generates AI-powered content. These two engines work in concert to ensure no Coder is left behind.

## User roles

Kairo serves two distinct user roles:

- **Coder** — A student enrolled in Riwi's bootcamp. Coders receive adaptive learning plans, focus cards, and access to an AI trainer chat tailored to their technical and soft skills profile.
- **Team Leader (TL)** — A mentor or instructor responsible for a cohort of Coders. TLs have access to an analytics dashboard that surfaces at-risk students and tracks cohort-wide progress before academic failure occurs.

## Backend engines

Kairo uses a hybrid architecture built on two independent backend services:

- **Node.js / Express 5 (API Gateway — port 3000):** The System Orchestrator. Manages user authentication (including GitHub and Google OAuth), session handling, business logic, EJS server-side rendering, and all communication between the frontend and the Python AI service.
- **Python / FastAPI (AI Brain — port 8000):** The AI Microservice. Handles local LLM inference via Groq, embedding generation with `sentence-transformers` (`all-MiniLM-L6-v2`), personalized plan generation, focus card creation, AI trainer chat, and TL report generation.

<Note>
  Kairo's hybrid architecture separates concerns deliberately: Node.js owns security, sessions, and data orchestration, while Python owns all AI computation. This means the AI service can be scaled, swapped, or upgraded independently without touching the core business logic layer.
</Note>
