# Focus cards

Focus cards are six AI-generated study cards produced from your active learning plan. Each card targets a specific topic and tells you exactly what to read, what to practise, and what to watch out for — adapted to your learning style and soft skills profile.

## Distribution

Every generation produces exactly **6 cards** with a fixed priority distribution:

| Priority | Cards |
|----------|-------|
| High | 2 |
| Medium | 2 |
| Low | 2 |

High-priority cards target the topics most likely to block your progress. Low-priority cards reinforce foundational concepts that support the rest of your learning.

## Card structure

Each card contains the following fields:

| Field | Type | Description |
|-------|------|-------------|
| `priority` | `"high"` \| `"medium"` \| `"low"` | Urgency level |
| `topic` | string | The topic name this card addresses |
| `title` | string | A specific, descriptive card title |
| `theory` | string | A 3–4 sentence explanation of the concept |
| `practice.description` | string | What you should do to practise |
| `practice.example` | string | A code snippet or step-by-step walkthrough |
| `pro_tips` | string[] | A list of tips from Kairo's experience with this topic |

## How to request focus cards

Focus cards are generated by sending a `POST` request to the Python AI service. The Node.js backend at `POST /api/ia/generate-focus-cards` assembles the payload and forwards it.

```python
{
  "coder_id": 42,
  "module_id": 3,
  "struggling_topics": ["async/await", "closures", "event loop"]
}
```

If `struggling_topics` is empty, Kairo falls back to `"general programming concepts"` for the current module.

<Note>
  Focus cards are generated by `POST /generate-focus-cards` on the Python AI service (port 8000). The Node.js API at `/api/ia/generate-focus-cards` acts as a proxy that resolves topic IDs to names before forwarding the request.
</Note>

## What makes focus cards adaptive

<AccordionGroup>
  <Accordion title="Learning style adaptation">
    The AI prompt receives your `learning_style` value from the `soft_skills_assessment` table. A `kinesthetic` learner receives cards with more hands-on code examples and fewer abstract explanations. A `visual` learner receives cards that suggest diagram-based approaches. A `mixed` learner receives a balanced blend.
  </Accordion>
  <Accordion title="Autonomy and problem solving scores">
    Your `problem_solving` score (1–5) and `autonomy` score (1–5) are included directly in the AI prompt. A coder with low autonomy (1–2) receives more guided, step-by-step practice instructions. A coder with high problem-solving ability (4–5) receives more open-ended challenges with fewer hints.
  </Accordion>
  <Accordion title="Struggling topics from Moodle">
    The `struggling_topics` array is populated from your `moodle_progress` row in the database. These are the specific topic names where your Moodle performance data signals difficulty. High-priority cards are assigned to your most critical struggling topics first.
  </Accordion>
  <Accordion title="Module context">
    The `module_id` tells the AI which part of the bootcamp curriculum you are currently studying. Cards stay within the scope of your active module — they will not surface topics from modules you have not reached yet.
  </Accordion>
</AccordionGroup>

## Example card

```json
{
  "priority": "high",
  "topic": "async/await",
  "title": "Handling asynchronous operations with async/await",
  "theory": "async/await is syntactic sugar over Promises that lets you write asynchronous code in a synchronous style. The async keyword marks a function as returning a Promise. The await keyword pauses execution inside that function until the awaited Promise resolves. Errors thrown inside an async function reject the returned Promise and can be caught with try/catch.",
  "practice": {
    "description": "Rewrite a callback-based function using async/await and handle errors correctly.",
    "example": "async function fetchUser(id) {\n  try {\n    const res = await fetch(`/api/users/${id}`);\n    if (!res.ok) throw new Error('Not found');\n    return await res.json();\n  } catch (err) {\n    console.error(err.message);\n  }\n}"
  },
  "pro_tips": [
    "Always use try/catch inside async functions — unhandled rejections crash Node processes.",
    "await only works inside async functions. Using it at the top level requires a module context."
  ]
}
```
